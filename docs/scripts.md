# Scripts Reference

Documentation for the helper scripts in the `scripts/` directory.

---

## `scripts/tf` — OpenTofu wrapper

The main entry point for running OpenTofu commands against this project. It handles credential loading, variable file discovery, and backend configuration automatically so you don't have to pass flags manually.

### What it does

1. **Loads environment files** — finds all `*.env` and `*.env.sh` files in the working directory and sources them into the shell before invoking `tofu`. This is how OTC credentials (`OS_ACCESS_KEY`, `OS_SECRET_KEY`, `OS_PROJECT_NAME`) are picked up.
2. **Auto-discovers var files** — passes every `*.tfvars` file in the working directory as a `-var-file` argument.
3. **Derives the region** — extracts the region from `OS_PROJECT_NAME` (e.g. `eu-de_myproject` → `eu-de`) and sets `OS_REGION_NAME`, `OS_AUTH_URL`, and `TF_VAR_region` automatically.
4. **Maps OTC credentials to AWS env vars** — copies `OS_ACCESS_KEY` / `OS_SECRET_KEY` to `AWS_ACCESS_KEY_ID` / `AWS_SECRET_ACCESS_KEY` so the S3-compatible remote backend works without a separate credential block.
5. **Handles `init` specially** — on `tofu init` it also passes `backend.hcl` as `-backend-config=backend.hcl`.
6. **Skips var files for state/workspace commands** — commands like `fmt`, `state`, `workspace` do not accept `-var-file`; the wrapper detects these and calls `tofu` directly without var-file flags.

### Usage

```
scripts/tf [options] <tofu-command> [tofu-args...]
```

**Options** (must come before the tofu command):

| Flag | Description |
|---|---|
| `-e <dir>` / `--vars-dir=<dir>` | Look for env and var files in `<dir>` instead of `.` |
| `-chdir=<dir>` / `-c <dir>` | Change to `<dir>` before running |
| `-h` / `--help` | Print tofu help |
| `-V` / `--version` | Print tofu version |

### Examples

```bash
# Initialise — downloads providers and configures the S3 backend
./scripts/tf init

# Show what would change
./scripts/tf plan

# Apply changes
./scripts/tf apply

# Destroy a specific module only
./scripts/tf destroy -target=module.apps

# Use var files from a different directory
./scripts/tf -e /path/to/env plan

# Run from a different project directory
./scripts/tf -c ../other-project plan
```

### Credential file (`*.env`)

Create a file named `dev.env` (or any `*.env` name) in the project root:

```bash
export OS_ACCESS_KEY="your-access-key"
export OS_SECRET_KEY="your-secret-key"
export OS_PROJECT_NAME="eu-de_your-project"
```

The wrapper sources this automatically on every invocation. The file is excluded from `pack` and `git` (see `.gitignore`) to avoid accidental credential leaks.

## `scripts/pack` — source archive

Creates a zip archive of the project source code, intentionally excluding all secrets and generated files. Useful for sharing or archiving the Terraform code without risk of leaking credentials or state.

### Excluded files

| Pattern | Reason |
|---|---|
| `*.tfvars` | Contain secrets (passwords, keys) |
| `backend.hcl` | Contains bucket name / access details |
| `*.key`, `*.crt`, `*.pem` | TLS private keys and certificates |
| `terraform.tfstate*` | State files — can contain secrets |
| `*.env`, `*.env.sh` | Credential files |
| `.terraform/` | Provider binaries — large, reproducible via `init` |
| `.terraform.lock.hcl` | Lock file — regenerated by `init` |
| `tmp.*` | Temporary files |

### Usage

```
scripts/pack [options] <zipfile>
```

| Flag | Description |
|---|---|
| `-t` | Dry run — print the file list without creating the zip |
| `-d <dir>` / `--chdir <dir>` | Pack from `<dir>` instead of `.` |
| `-` (as zipfile) | Write file list to stdout instead of a zip |

### Examples

```bash
# Preview what will be included
./scripts/pack -t

# Create an archive
./scripts/pack /tmp/iaas-demo.zip

# Pack a different directory
./scripts/pack -d ../other-project /tmp/other.zip
```

## `scripts/xvars` — secrets archive

Creates a zip archive containing *only* the secret/config files that `pack` excludes. The complement to `pack` — together they cover the full project.

### Included files

| Pattern | Contents |
|---|---|
| `*.tfvars` | Variable files (passwords, endpoints, keys) |
| `backend.hcl` | Remote state configuration |
| `*.pem`, `*.crt`, `*.key` | TLS material |
| `*.env`, `*.env.sh` | Credential env files |

`inputs.tfvars` is always excluded (it is treated as a generated/intermediate file).

### Usage

```
scripts/xvars [options] <zipfile>
```

| Flag | Description |
|---|---|
| `-l` / `-t` / `--list` | List files with `ls -lh` instead of creating a zip |
| `-s` / `--links` | Include symlinks in the archive (default) |
| `-S` / `--no-links` | Exclude symlinks |
| `-d <dir>` / `--chdir <dir>` | Export from `<dir>` instead of `.` |
| `-` (as zipfile) | Write file list to stdout |

### Examples

```bash
# List which secret files exist and their sizes
./scripts/xvars --list

# Export secrets to a zip for safe storage/handover
./scripts/xvars /tmp/iaas-demo-secrets.zip

# Export without following symlinks
./scripts/xvars --no-links /tmp/iaas-demo-secrets.zip
```

## `scripts/dns2ip` — DNS lookup for Terraform

Resolves a DNS name to an IP address and prints the result as a JSON object that Terraform's `external` data source can consume.

This script is intended to be called from a Terraform `data "external"` block to look up an IP at plan/apply time when an IP address is not directly available as a resource attribute.

### Output format

```json
{ "value": "10.1.2.3" }
```

If the name does not resolve, it returns the default (`0.0.0.0` unless overridden).

### Usage

```
scripts/dns2ip <dns_name> [default_ip]
```

| Argument | Description |
|---|---|
| `dns_name` | Hostname or FQDN to resolve |
| `default_ip` | Value to return if resolution fails (default: `0.0.0.0`) |

### Examples

```bash
# Resolve a name
./scripts/dns2ip bastion.example.public.t-cloud.com

# Resolve with a custom fallback
./scripts/dns2ip bastion.example.public.t-cloud.com 127.0.0.1
```

### Terraform usage

```hcl
data "external" "peer_ip" {
  program = ["${path.module}/../../scripts/dns2ip", "peer.example.com"]
}

# Access the result:
# data.external.peer_ip.result["value"]
```
